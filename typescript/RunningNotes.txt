        Forms in React 19:
            React 19 introduces a more standardized way to handle forms using new features like 'Actions', 'useActionState' and 'useFormStatus' hook.
            These simplify form submission, loading states and error handling

React 19 Forms: Form handling involves using 'useState()' to track input values writing 'onChange' handles for each field handling submission via handleSubmit() function

React 19 Form Enhancements:
    React 19 introduces Actions, which are asynchronous functions designed to simply form logic.
    Handle form submission declaratively, track pending states with 'useFormStatus' manage form state within 'useActionState()'

Actions are asyn functions that handle form logic we pass them directly to the forms  action attribute.

    Key features in React 19 forms  
        Actions -> These are asynchronous functions that handle form submissions. Instead of manually managing state and side effects, we define an action function and pass to the form.

        'useActionState' => It is a new hook that has been introduced in react 19, this hook tracks the state of an action(eg. loading,success,error). It replaces manual 'useState' and 'useEffect' logic form submission
         UseAction State Syntax:
    Takes 2 paramameters
        1.Form SUbmit handler function to handle logic after user clicks on button
        2.Object to specify the initial state of form


        'useFormStatus' -> 'its a new hook that has been introduced from react 19. This hook provides realtime status of the form(eg:pending, submitted). It is useful for disabling buttons or showinf spinners during submission.
        Note:
        We generally dont need to depend on 3rdparty libary like Formik from react q9, React 19 introduces native form handling features that reduces the need for external library like formik or React hook form.

    useContext() -, createContext()

        useContext() is a predefined hook which is supported in react, using which we could 
        be able to access / subscribe the context memory directly within the funcitonal component.

        It works with a context object created using React.createContext()

        Context memory is a special memory of every component, which can be accessible only to other child components.

        Note: The context memory will not be created for every component by default.

        Steps to create context memory within a component

           1. var ContextName = createContext(null); //Create a context memory

           2. define scope of context memory
            -> <Contextname.provider>
                <ChildComp></ChildComp>
               </Contextname.provider>

            3. Accessing context memory data under child
                -> let obj = useContext(contextname);

            ".Provider" : In react Provider is a special component which comments
            with every object created using React.createContext();
                Its used to provide a value to all components inside its tree,so they can access that value
                via :"useContext" without prop drilling
                '.Provider' is essentially the gateway for passing data down the component tree

            Note:Adding .Provider is optional from lastest React

            Note:

                1. By default none of the conponents hold the context memory
                2.Context memory only gets created by using createContext()
                3.We can create multiple context memory for a single component
                4.Yes child components can access data from different context memory, through useContext("contextname)
